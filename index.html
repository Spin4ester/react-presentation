React.js

A JavaScript library for building user interfaces

It was created in 2013 by Facebook during the process of integrating chat to their social network

Used for handling the view layer and can be used for web and mobile apps.

Library

  • Сollection of helper modules, objects, classes, functions, message templates, pre-written code and etc.
  • Flexbility in modifying and calling.
  • Designed to accomplish a specific purpose.
  • Ability to integrate into existing project.

Framework

  • Foundation upon which developers build applications for specific platforms.
  • Frameworks consist of a lot of APIs, compilers, toolsets, support programs, libraries, etc.
  • Limited flexibility
  • Incorporating frameworks seamlessly into an existing project is close to impossible.

Difference

Inversion of control

Library is like building a house, framework is buying a house

Virtual DOM

Manipulating DOM with JavaScript is slow

Virtual DOM is a lightweight copy of the actual DOM

  1. Changes are made in Virtual DOM
  2. Virtual DOM is being compared to the actual DOM (object with differences is built)
  3. React updates the actual DOM with maximal efficiency

Components

Piece of the UI that has its own logic and appearance.

Components are like functions that return HTML elements.

Can be as small as a button, or as large as an entire page.

All components must act like pure functions with respect to their props.

Functional Components

No state of their own and contain only a render method.


							function Welcome(props) {
                return 
Hello, {props.name}
;
              }

Class Components

Class components have a built-in state object and are more complex than functional components.

Requires a render() method that returns HTML.

Have access to a state which dictates the current behavior and appearance of the component 


							class Welcome extends React.Component {
                render() {
                  return 
 Hello, {this.props.name}
;
              }
            }

State

The state object stores property values that belongs to the component.

Whenever state changes, the component re-renders.

Multiple properties can be stored in the state object.


            class Car extends React.Component {
              constructor(props) {
                super(props);
                this.state = {brand: "Ford"};
              }
              render() {
                return (
                  

                  
My car is {this.state.brand}

                  

                );
              }
            }

Props

Props stand for properties, and they are read-only components.

Props are arguments passed into React components.

Always passed down from the parent component to the child component.

Similar to function arguments in JavaScript and attributes in HTML.
